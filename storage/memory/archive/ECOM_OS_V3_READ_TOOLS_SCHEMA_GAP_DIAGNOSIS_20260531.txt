The still-problematic tools are read tools such as:
- read_file_safe

This chat had the archive content already in conversation context and also had successful write confirmations. It could reason from previously saved outputs and user-supplied server outputs. That is not the same as direct read_file_safe(path) access.

WHY NEW CHAT FAILS:
New chat starts without this chat's long context. It relies on connector tools. If read_file_safe has no explicit inputSchema for path, it cannot pass archive path to the MCP service. github_memory_read_latest_archive may return an older latest pointer and not the specific file Ion asks for.

Partial MCP schema repair. Write schema fixed; read schema still incomplete.
