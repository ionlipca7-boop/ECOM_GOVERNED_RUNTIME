This contract upgrades the previous loose GitHub-memory route drafts into a strict server-ready contract for all 8 Telegram route-blocks. It must be used as the only basis for the later unified server transfer packet.

No Telegram restart.
No eBay call.

## V7.3C Route Architecture Rule

8 main Telegram buttons = 8 independent route-blocks.

Each route-block owns only its own:
- route_id
- callback namespace
- case namespace
- case state
- screens
- agents
- reports
- schemas
- safety gates
- repair path

Shared agents may advise, but no shared agent may own route state.
Route modules may link/handoff to each other only through explicit handoff objects.
No route imports/includes another route directly in a circular way.
The future root router may attach all route modules, but route modules must not own the root router.

## Callback Registry Rules

All callbacks must:
- be unique globally;
- be 1-64 bytes UTF-8;
- start with v73c:;
- use route-specific namespaces: r01, r02, r03, r04, r05, r06, r07, r08;
- include route_home callback: v73c:route:<route_slug>;
- include back/home/stop for every route.

## Global Navigation Contract

Back:
- returns to previous screen inside current route only;
- no delete;
- no publish;
- no eBay;
- no route closure.

Home:
- opens main menu only;
- does not show paused cases;
- if current case unfinished, mark PAUSED and create Route 07 decision_card.

Stop:
- asks confirmation;
- options: cancel case or leave paused;
- no immediate physical delete;
- cleanup only through cleanup gate.
