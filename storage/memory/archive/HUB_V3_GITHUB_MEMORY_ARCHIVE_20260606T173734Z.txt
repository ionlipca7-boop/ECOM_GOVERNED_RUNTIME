## Purpose

Route 01 handles one simple product / one SKU / one listing draft. Default target is eBay.de, but the core must remain marketplace-neutral through product_passport + marketplace_adapter_contract.

Route 01 must take raw Ion input and turn it into a safe, reviewable product package:
- source link/photos/receipt/specs/comments;
- product_passport;
- photo_plan;
- listing_content;
- item specifics / attributes;
- price/margin;
- stock/finance link;
- compliance_packet;
- eBay offer readiness packet;
- approval gate for publish.

Route 01 must not publish automatically.

## What Bot Does From Start To Finish

Step 0 — Open Route 01
Bot shows Route 01 home screen:
"Пришли ссылку, фото, чек, характеристики, комментарии. Можно всё в любом порядке. Я буду молча собирать кейс."
Creates route_state with status NEW/COLLECTING.

Step 1 — Silent Intake
Bot accepts:
- supplier/product link;
- product photos;
- receipt/order screenshot;
- product specs text;
- Ion comments;
- package/quantity notes;
- target price/strategy if Ion gives it.

Step 2 — Accepted Inputs Report
Bot shows:
- link: yes/no;
- photos count;
- receipt: yes/no;
- specs: yes/no;
- comments: yes/no;
- current missing fields;
- route_state status.

Step 3 — Build Product
Agents build product package in dry mode.
No eBay API call.
No marketplace write.

Step 4 — Show Short Product Card
Bot shows:
- product name;
- source;
- category candidate;
- cost;
- quantity;
- main value signal;
- risk flags;
- next action.

Step 5 — Show Full Listing In Parts
Bot never sends giant wall of text. It shows sections:
1. product card;
2. title + price;
3. product description;
4. designed HTML/description block;
5. item specifics / attributes;
6. photo plan;
7. shop / cross-sell block;
8. final draft summary.

Step 6 — Rework
Ion can rework title, description, price, specifics, photo plan, shop block, keywords, whole listing, or custom text correction.

Step 7 — Readiness Check
System checks title/photos/category/aspects/description/price/quantity/policies/duplicate/compliance/finance.
If missing -> blocks and returns exact repair list.
If PASS -> publish gate packet is allowed.

Step 8 — Publish Gate
Bot shows approval packet only.
No eBay publish without separate marker.

Step 9 — After Future Publish
Future only: successful publish creates handoff to Route 04 lifecycle and Route 05 stock/finance.

## Route 01 Agent Team

### 01 Intake Agent
Purpose: collect raw inputs in any order.
Inputs: Telegram messages, photos, links, receipt screenshots, Ion notes.
Outputs: accepted_input_state.
Rules: must not ask too many questions; silent collection first, missing fields later.

### 02 Source Link Agent
Purpose: identify product source and supplier truth.
Inputs: supplier link, product page screenshots, title, price, specs.
Outputs: source_snapshot, supplier_truth_notes.
Rules: no blind trust; supplier claims are unverified until evidence exists.

### 03 Photo Intake + Truth Agent
Purpose: organize photos and detect whether photos are real product/supplier/edited.
Inputs: Telegram photos, source images.
Outputs: photo_manifest, photo_truth_report.
Rules: product geometry must not be invented.

### 04 Receipt / Cost Agent
Purpose: extract purchase cost, quantity, shipping cost, order date.
Outputs: receipt_cost_record.
Rules: unknown cost blocks final profit calculation.

### 05 Product Passport Agent
Purpose: create master product identity.
Outputs: product_passport.
Fields: master_product_id, SKU candidate, name, brand/model, product type, package truth, dimensions if known, electrical/spec fields if relevant, supplier link, photo refs, risk flags.

### 06 Category Agent
Purpose: choose marketplace category candidate.
Default: eBay.de.
Outputs: category_candidate_report.
Rules:
- no hardcoded category as final truth;
- category must be derived from product type + marketplace taxonomy/aspects;
- if uncertainty -> NEEDS_REVIEW;
- future Amazon/Kaufland/OTTO category/taxonomy handled by adapters.

### 07 eBay Aspect / Item Specifics Agent
Purpose: produce item specifics required/recommended for eBay category.
Outputs: item_specifics_draft, missing_aspects_report.
Rules:
- must include category-driven required aspects when available;
- must use confirmed facts only;
- unknowns stay unknown, not invented.

### 08 Title SEO Agent
Purpose: create German eBay title.
Rules:
- max eBay title length: 80 characters;
- use buyer search + value signal;
- no false brand/model/features;
- for technical products use formula: Brand/Model if confirmed + product class + premium feature + value function + buyer intent + protocols.
Outputs: title_draft, title_score, keyword_notes.

### 09 Description Agent
Purpose: create German product description.
Outputs: plain_description, designed_description_html.
Rules:
- clear German;
- buyer benefit first;
- package contents exact;
- avoid false claims;
- mobile short summary must be clear.

### 10 Photo Plan Agent
Purpose: create eBay photo plan.
Rules from eBay picture policy:
- at least one photo required;
- photo must be at least 500 px on longest side;
- no text/artwork/marketing material/borders/watermarks;
- photos must accurately represent item;
- hero photo should be clean and product-truth based.
Project rule:
- source-based edit only;
- no invented product geometry;
- hero no text/watermark;
- additional images may explain only if compliant and not misleading.
Outputs: photo_plan, photo_rework_tasks.

### 11 Compliance / GPSR Agent
Purpose: evaluate EU/Germany product safety/compliance risk before listing.
Outputs: compliance_packet.
Fields: GPSR info requirement, manufacturer/importer/responsible person if needed, warnings, safety info, CE/EPR/WEEE/battery flags, product risk level.
Rules:
- if unsafe/uncertain -> NEEDS_REVIEW;
- compliance issue blocks readiness.

### 12 Stock Link Agent
Purpose: connect product to inventory/stock pool.
Outputs: stock_link_record.
Fields: SKU, master_product_id, quantity bought, quantity available, linked marketplace listing IDs later.

### 13 Finance / Pricing Agent
Purpose: calculate profitable price.
Inputs: purchase cost, China shipping, Germany shipping, packaging, eBay fee estimate, ad %, return/defect reserve.
Outputs: price_margin_report.
Rules:
- no profit PASS without return/defect reserve;
- default ad estimate can be 6-8% or project setting;
- unknown costs block final readiness.

### 14 eBay Offer Readiness Agent
Purpose: convert product draft into eBay Inventory API readiness packet.
Outputs: ebay_simple_offer_readiness.
Needed for future live phase:
- SKU;
- marketplaceId EBAY_DE;
- format fixed price;
- inventory location;
- available quantity;
- categoryId;
- price;
- payment policy;
- fulfillment policy;
- return policy;
- item specifics/aspects;
- image URLs;
- product/offer description;
- compliance fields when required.
Rules:
- createOffer stages offer only; publishOffer is separate live action;
- publish gate must remain locked.

### 15 Duplicate Guard Agent
Purpose: avoid duplicate listing mistakes.
Outputs: duplicate_risk_report.
Rules:
- compare SKU/product/title/source/listing_id;
- if same physical item already has listing, send to Route 03 or update gate, not clone.

### 16 Readiness Agent
Purpose: final dry readiness verdict.
Outputs: readiness_report.
Verdicts:
- READY_FOR_REVIEW
- NEEDS_REWORK
- BLOCKED_MISSING_DATA
- BLOCKED_COMPLIANCE
- BLOCKED_DUPLICATE
- BLOCKED_FINANCE
- READY_FOR_EBAY_GATE

### 17 Publish Gate Agent
Purpose: prepare approval packet only.
Outputs: publish_gate_packet.
Rules:
- no eBay live/publish/revise/delete without marker;
- no token print;
- no hidden live write.

### 18 Handoff Agent
Purpose: move cases to other routes only through handoff object.
Examples:
- variants detected -> Route 02 handoff;
- weak draft -> Route 03 handoff;
- ready stock/cost -> Route 05 handoff;
- paused/block/approval -> Route 07 handoff;
- archive/lesson -> Route 08 handoff.
