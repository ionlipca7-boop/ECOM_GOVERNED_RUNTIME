## Official + practical audit applied

Telegram:
- callback_data must stay 1-64 bytes.
- callback_query must be handled as update type.

Ebay photos:
- at least one image required.
- longest side at least 500 px.
- no inaccurate item representation.
- no placeholder/message images.
- no borders.
- no added text/artwork/marketing material.
- no watermarks.

Ebay inventory item:
- SKU is unique per seller inventory.
- SKU max length 50.
- Inventory API listings cannot be revised/relisted with Trading API.
- Product title max length 80.
- Product description supports basic HTML but no active content.
- Item aspects are needed before publish.
- Inventory item replacement is complete replacement, so read current item before future update.

Ebay offer:
- createOffer only stages offer.
- createOffer initially requires sku, marketplaceId, listing format.
- Before publish, location, price, quantity, category and business policies become required.
- publishOffer is separate live step.
- regulatory/GPSR product safety and responsible person data may be conditionally required.

Ebay taxonomy/aspects:
- getItemAspectsForCategory returns category aspects, required/recommended status, variation support, allowed values, data type, cardinality, and buyer demand relevance if access exists.
- Category must be leaf category for aspect check.

Route 02 future reuse note:
- eBay variations use inventory item group, variantSKUs, variesBy, aspectsImageVariesBy, specifications.
- Therefore Route 01 handles one SKU only; Route 02 must extend, not duplicate, Route 01 base logic.

Practical seller guidance applied as non-policy inspiration only:
- multiple clear photos;
- honest description;
- good lighting;
- competitive/realistic pricing;
- buyer trust and communication.
Official eBay policy remains authority.
