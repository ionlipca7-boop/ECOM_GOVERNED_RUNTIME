## Route 01 agent updates from screenshots

### Photo Plan Agent update
Must understand eBay UI supports many photos/video, but official photo policy still controls.
Project target for new listings:
- minimum: 6 useful photos if available;
- better: 8-12 high-quality truthful photos;
- maximum planning: up to 24 photos + 1 short video if available;
- hero image clean, no text/watermark;
- secondary images can show features only if compliant and not misleading.

### Category Agent update
For charger/cable product type, observed category path:
Kabel & Adapter in Handys & Kommunikation > Handy-Zubehör.
Rule: observed category can be used as candidate, not as final hardcoded category. Must verify by product type and eBay category/taxonomy.

### eBay Aspect Agent update
Must separate:
- required fields: Marke, Produktart;
- optional high-demand fields;
- physical dimension fields;
- personalization fields;
- guarantee/condition fields.
If EAN missing, set Does not apply / Nicht zutreffend only if marketplace/category permits; otherwise NEEDS_REVIEW.

### Title SEO Agent update
Title must use high-signal fields where truthful:
- product type
- connector/connectivity
- power/watt
- cable included if true
- fast charging / PD if true
- buyer function.
But no unconfirmed brand/spec.
Max 80 characters.

### Description Agent update
Description must produce:
- full German description;
- mobile-friendly summary;
- bold headings/sections;
- package contents exact;
- use case / buyer pain / benefit;
- compatibility limits;
- safety notes if needed.
AI-description may be used only as draft inspiration, never as final truth without verification.

### Finance/Pricing Agent update
Must read UI price suggestions as a signal:
- sold offers last 90 days recommended price;
- free-shipping share;
- current price;
- ad rate.
But final price must come from cost + fee + reserve + market signal.

### eBay Offer Readiness Agent update
Must map UI fields to API readiness:
- title -> product.title / offer listing title source;
- category -> offer.categoryId;
- condition -> inventory item condition;
- item specifics -> product.aspects;
- description -> offer.listingDescription or inventory product.description;
- price -> pricingSummary.price;
- quantity -> availableQuantity / inventory availability;
- policies -> listingPolicies payment/fulfillment/return;
- package size -> packageWeightAndSize;
- product documents/GPSR -> compliance/product docs/custom policy layer when applicable;
- promoted listing -> not part of publish core; separate marketing gate.

### Compliance/GPSR Agent update
Must include UI product documents section:
- manuals;
- certificates;
- safety/quality standards;
- accessibility information;
- EU/UK delivery note.
For electronics/chargers/cables this becomes more important than for low-risk simple items.

### Publish Gate Agent update
Publish gate packet must include a UI checklist:
- photos complete;
- title complete;
- category confirmed;
- variants warning checked;
- required item specifics filled;
- high-search optional specifics reviewed;
- condition set;
- description checked;
- price/quantity set;
- payment/fulfillment/return policies set;
- shipping details/package size checked;
- product documents/compliance checked;
- ad/promoted listing decision checked;
- preview checked;
- no live call until marker.

## Route 01 official rules to preserve

Photo policy:
- at least one image required;
- image at least 500px on longest side;
- no inaccurate photos, placeholders, stock photos for used/damaged/defective items, borders, added text/artwork/marketing material, or watermarks.

Inventory item:
- SKU unique across seller inventory;
- SKU max length 50;
- Inventory API listings must not be revised/relisted with Trading API;
- Product title max length 80;
- product aspects/images/description are product-layer fields;
- replacing inventory item is full replacement, so read current item first before future updates.

Offer:
- createOffer is staging;
- publishOffer is separate;
- before publish, category, location, price, quantity, listing policies and description are required/conditional according to state.

Taxonomy/aspects:
- category must be leaf for aspect checks;
- aspects can be required/recommended and may have allowed values, cardinality, data type and variation support.

## Route 01 practical seller guidance to use carefully

Non-official practical guidance confirms useful but not authoritative rules:
- many clear photos build buyer trust;
- good lighting and clean background help;
- title should put important buyer terms early;
- accurate, detailed description reduces returns;
- sold/completed listings help price realistically;
- tracking, packaging and communication affect buyer satisfaction.

Practical guidance is inspiration only. Official policy and product truth remain authority.
