## Key extraction from uploaded chat for Route 01

The uploaded chat defines ECOM OS Market Intelligence Chain V2.1R with Return Reserve and Product Radar logic. Useful Route 01 parts are:

1. Return Reserve rule.
A product is not profitable until it survives return reserve and defect reserve.
Route 01 pricing must calculate profit before reserve and profit after reserve before any publish gate.

2. Value Signal rule.
Title alone is not enough. A listing becomes strong through:
Title + Item Specifics + Photos + Description + Package Truth + Delivery / Ads logic.
Route 01 must generate all of these as one connected system, not as isolated text blocks.

3. UD24 title/value lesson.
Technical products need confirmed value words and product function words, not generic low-value titles.
Formula:
Brand/Model if confirmed + Product Class + Premium Feature + Value Function + Buyer Intent + Protocols if true.
This should guide R01 German Title SEO Agent and Search Demand Aspects Agent.

4. Buyer search logic.
Route 01 should use two input types from Product Radar:
Problem Search and Product Search.
Problem Search asks what pain buyer has.
Product Search asks what exact product buyer searches.
For Route 01, these signals are used in title, item specifics, description and photo plan.

5. Offer Structure awareness.
Even for a single product, Route 01 must check whether it is actually:
- single listing;
- variation candidate;
- bundle candidate;
- replacement listing candidate;
- premium/niche listing candidate;
- watch-only candidate.
If it is not truly one simple product, Route 01 hands off to Route 02 or Route 03/06/07.

6. No duplicate listing rule.
Each listing must have distinct buyer intent. If a proposed Route 01 listing duplicates an existing product/listing, it must go to Route 03 Listing Studio or Route 07 decision, not create a clone.

7. Supplier evidence rule.
Route 01 must not trust supplier page blindly. It must preserve supplier claims as unverified until evidence exists: screenshots, photos, specs, receipt, reviews, or Ion confirmation.

8. Manual input route.
Ion can send photo, link, screenshot, supplier listing, product name or idea. Route 01 intake should accept all these and build an accepted_input_state.

9. Missing data route.
If real supplier/browser/scraper is not available, Route 01 can still build a draft from screenshots/manual evidence, but must mark confidence and missing evidence.

10. Learning loop.
Every Route 01 listing draft and later result must create lessons for Route 08 / Master Teacher: title worked/not worked, price worked/not worked, photo/aspect missing, return/defect assumptions, eBay recommended price movement.

Reserve defaults from product-search chat:
- simple goods: 5-7 percent return reserve + 2-3 percent defect reserve;
- compatibility risk goods: 8-10 percent return + 2-4 percent defect;
- electronics/chargers/Bluetooth/complex compatibility: 12-15 percent+ return + 3-5 percent+ defect.

Route 01 charger/cable products should start with electronics/compatibility reserve until evidence reduces risk.
