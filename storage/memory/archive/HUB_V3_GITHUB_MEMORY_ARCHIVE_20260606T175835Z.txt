## Route 02 Purpose

Route 02 handles a master product with variants. It extends Route 01 but does not duplicate Route 01.

Route 02 is used when a product has selectable choices such as:
- color;
- size;
- length;
- kit size;
- pack count;
- power/wattage;
- capacity;
- connector type;
- model;
- material;
- bundle option;
- version/generation.

Goal:
Turn a multi-option product into a clean variant listing candidate with master product truth, variant matrix, SKU map, photo map, price/quantity map, item specifics/aspects, compliance, profit after reserve, and publish gate packet.

Route 02 must prevent partial/incorrect variation listings.

## Why Route 02 must be separate from Route 01

From Ion's real eBay UI screenshot:
If the original listing did not include variants, eBay shows that variants are not available and a new listing/offer is needed for variants. Therefore variants must be detected before publish or update, not after.

From eBay Inventory API:
Multiple-variation listings require inventory item groups. The group contains variantSKUs and variesBy specifications. Publishing a group is done through publishOfferByInventoryItemGroup, not normal publishOffer.

Route 01 handles one SKU.
Route 02 handles master product + multiple SKUs + inventory item group readiness.

## Route 02 Search Demand + Value Signal Rules

Route 02 inherits Route 01 search-demand logic and adds variant-specific logic:
- high-demand aspects can become variation dimensions only if truthful;
- if eBay UI shows search count/trend for aspect such as color, length, connectivity, pack count, those aspects should be prioritized in variant matrix and listing content;
- title must not list every variant if too long; use master offer + strongest variant dimension;
- description and item specifics must clarify all options;
- photo plan must prove variant differences;
- price/quantity must be clear per variant;
- no variant should hide weak margin.

## Route 02 Offer Structure Rules

Route 02 is correct if buyer sees options as one product family.

Route 02 is wrong if:
- options are unrelated products;
- variants have different buyer intent;
- bundle/kit logic is more important than selectable variant logic;
- marketplace category does not support intended variation aspects;
- variant matrix is incomplete.

If wrong, handoff:
- Route 03 for listing strategy / separate buyer intents;
- Route 06 for product radar / offer structure decision;
- Route 07 for Ion decision.
