## Official Rules Applied

Ebay Inventory API:
- multiple-variation listings use inventory item group;
- group uses inventoryItemGroupKey, variantSKUs, variesBy specifications and optionally aspectsImageVariesBy;
- group title max 80;
- group description becomes the multiple-variation listing description;
- group updates are complete replacements, so future updates must read the current group first;
- group publish uses publishOfferByInventoryItemGroup and requires valid inventory items/offers.

Ebay Taxonomy:
- only aspects with aspectEnabledForVariations=true can be used as variation dimensions;
- aspects can be required, recommended or optional;
- aspects have type, cardinality, max length, allowed values and mode.

Ebay Picture Policy:
- at least one image required;
- image at least 500 px on longest side;
- no inaccurate photos, placeholder images, borders, added text/artwork/marketing material or watermarks.

Future marketplaces:
- Amazon needs product type / listing / restrictions adapter;
- Kaufland needs product data / variant suggestion / inventory adapter;
- OTTO needs its own OpenAPI/changelog adapter.

## Strict Route 02 Rules

- Route 02 is used only when options are one product family.
- If there are no real variants, handoff back to Route 01.
- If options are different buyer intents, send to Route 03 or Route 07.
- If market proof or supplier comparison is missing, send to Route 06 or Route 07.
- Every variant row needs option value, SKU status, EAN status, quantity, price and photo status.
- Price range must match min/max active variants.
- Total quantity must match sum of active variants.
- Variant images must match the selected image attribute and product truth.
- No guessed variant values.
- Any active variant with weak or negative profit after reserve blocks group readiness unless Ion approves strategic test.
